Feed
GET Feed.messages¶
Выдача сообщений ленты на основе параметрической фильтрации.

Максимальный лимит получения ленты сообщений - 150 сообщений.

Описание фильтров filter¶
Фильтры используются для определения выборки сообщений. В запросе предусмотрена возможность передавать сразу несколько фильтров.

Пример:

?filter[fm][1]=on&filter[fm][2]=on&filter[fauthor][a4936ef10690c3b1df4539b48a966e97]=on

В данном примере запрашивается лента с медиа типами 1 (Микроблоги) и 3 (Соцсети) и автор c hash a4936ef10690c3b1df4539b48a966e97 (Альфа-Банк).

Фильтр состоит из двух частей - названия фильтра ([fm],[fauthor]) и идентификатора объекта (Тип источника, Источник, Автор, ГЕО-информация, тональность)

filter[fts]	Время добавления сообщения в тему	Данный фильтр используется для выборки ленты в конкретном диапазоне времени добавления сообщений в тему.	Все
filter[ftexthash]	Хэш текста сообщения	Данный фильтр используется для выборки ленты по хэшу текста сообщения.	Все

filter[fduplicate]	Дубликаты	Данный фильтр используется для выборки ленты по дубликатам сообщения.	Соцмедиа+СМИ, Избранные источники, Избранные авторы, Сеты авторов
filter[fobscenities]	Ненормативная лексика	Данный фильтр используется для выборки ленты по ненормативной лексике. Всегда принимает значение 1. filter[fprodaspect]	Характеристики продуктов	Данный фильтр используется для выборки ленты по характеристиками продуктов.	Соцмедиа+СМИ, Избранные источники, Избранные авторы, Сеты авторов
Типы фильтрации.

Значение	Название	Описание
on	Включающая	Возвращает сообщения в которых есть хотя бы одно из перечисленных в фильтре значений.
off	Исключающая	Возвращает сообщения в которых нет ни одного из перечисленных в фильтре значений.
all	Пересечение	Возвращает сообщения в которых есть одновременно все из перечисленных в фильтре значений.

GET Feed.listMessagesCustom¶
Новый метод для получения всех сообщений из темы. Параметры и выдача аналогичны Feed.messages, только добавлен еще один входящий параметр fieldGroups, принимающий на вход массив названий групп полей.

Возможные группы	Описание	Макс сообщений в ответе	Лимит в минуту
dynamic	Динамически рассчитываемые поля	150	2
relational	Расширенные поля, которые требуют отдельных запросов	500	2
text	Текстовые поля сообщения	5000	6
static	Статические и неизменяемые поля	10000	12
counters	Поля, содержащие счётчики, некоторые могут со временем обновляться	10000	12
updatable	Поля, которые могут обновляться пользователем	10000	12
Возможно использовать группу updateable, которая является псевдонимом к группе updatable. Поддержка псевдонима updateable может быть отключена в последующих версиях API.

Пример¶
curl -XGET 'https://bans-api.brandanalytics.ru/ba_api/feed.listMessagesCustom?themeId=12384080&filter[fm][1]=on&timeFrom=1778274000&timeTo=1778878800&offset=0&limit=25&fieldGroups[]=static&fieldGroups[]=text&highlight=1'
