# История работы — ba-linni / Brand Analytics API
_Дата: 14 мая 2026_

---

## Цель

Подключиться к API сервиса Brand Analytics (brandanalytics.ru) с использованием ключа и загружать данные в папку проекта.

---

## Что было сделано

### 1. Создана папка `ba-linni`
Рабочая директория для проекта интеграции с Brand Analytics.

### 2. Поиск документации API
- Выполнен поиск публичной документации через WebSearch и WebFetch
- Найдено: Brand Analytics имеет «Статистический API», доступный с тарифа «Расширенный»
- Документация **не индексируется публично** — доступна только внутри личного кабинета
- Источник: [brandanalytics.ru/blog/statistical-api](https://brandanalytics.ru/blog/statistical-api)

### 3. Зондирование API (файл `test_api.js`)
Написан и запущен скрипт для автоматического перебора эндпоинтов и схем авторизации.

**Проверенные хосты:**
- `brandanalytics.ru` — все пути → 404
- `api.brandanalytics.ru` — корень `/` → **200** (пустая HTML-страница), все пути → 404
- `stat.brandanalytics.ru` — ENOTFOUND (поддомен не существует)

**Проверенные схемы авторизации:**
- `Authorization: Bearer <key>`
- `Authorization: Token <key>`
- `X-API-Key: <key>`
- `X-Auth-Token: <key>`
- Query params: `?token=`, `?api_key=`, `?apikey=`, `?key=`, `?access_token=`

**Результат:** Все варианты вернули 404. Хост `api.brandanalytics.ru` живой, но правильные пути неизвестны.

**Следующий шаг:** зайти в личный кабинет Brand Analytics →
`Настройки → Интеграции → Statistical API` → скопировать ссылку на документацию или базовый URL.
После этого скрипт для загрузки данных будет написан быстро.
